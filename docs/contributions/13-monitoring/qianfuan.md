## 遇到的问题和解决

1. 问题：前端日志需要持久化但 localStorage 空间有限
   解决：使用 sessionStorage 存储，限制最多 200 条记录，超出后自动轮转清除早期日志

2. 问题：指标收集不能影响正常业务流程
   解决：使用 try/finally 模式确保指标记录一定会执行，且监控逻辑本身轻量级，不影响用户体验

## 心得体会

通过本次监控配置任务，学习了前端可观测性的实现方式：

1. **结构化日志**有助于后续对接日志聚合平台（如 ELK），JSON 格式便于机器解析和搜索
2. **健康检查端点**是微服务架构的基础组件，前端通过主动探测后端健康状态可以在服务异常时提供更好的用户提示
3. **前端指标收集**虽然简单但实用，可以帮助开发和运维人员快速了解 API 调用质量
4. 良好的监控体系需要前后端协同，前端监控是对后端监控的补充而非替代
