# 物业管理系统 - API 接口设计文档

## 1. 接口基本说明
*   **Base URL:** `http://localhost:8080/api`
*   **请求格式:** `application/json`
*   **响应格式:** `application/json`
*   **认证方式:** Header 携带 `Authorization: Bearer {token}`

## 2. 统一响应格式
```json
{
  "code": 200,      // 200-成功, 401-未登录, 403-权限不足, 500-服务器错误
  "message": "操作成功",
  "data": null      // 具体业务数据（对象、数组或分页对象）
}
```

---

## 3. 核心 API 模块

### 3.1 身份认证模块 (Auth)
| 接口名称 | 方法 | 路径 | 描述 | 权限 |
| :--- | :--- | :--- | :--- | :--- |
| 用户登录 | POST | `/auth/login` | 提交账号密码，返回 JWT Token | 公开 |
| 获取当前用户信息 | GET | `/auth/me` | 获取当前登录用户的角色、姓名等 | 登录用户 |

### 3.2 房产管理模块 (House)
| 接口名称 | 方法 | 路径 | 描述 | 权限 |
| :--- | :--- | :--- | :--- | :--- |
| 获取房产列表 | GET | `/houses` | 分页查询（支持按楼栋、业主筛选） | 管理员 |
| 我的房产 | GET | `/houses/my` | 获取当前业主名下的所有房产 | 业主 |
| 添加房产 | POST | `/houses` | 登记新的房间信息 | 管理员 |
| 修改房产 | PUT | `/houses/{id}` | 修改房间面积、绑定业主等 | 管理员 |
| 删除房产 | DELETE | `/houses/{id}` | 删除房产记录 | 管理员 |

### 3.3 费用管理模块 (Bill)
| 接口名称 | 方法 | 路径 | 描述 | 权限 |
| :--- | :--- | :--- | :--- | :--- |
| 账单查询 | GET | `/bills` | 分页查询所有账单及缴费状态 | 管理员 |
| 我的账单 | GET | `/bills/my` | 业主查询自己未缴/已缴的费用 | 业主 |
| 生成账单 | POST | `/bills/generate` | 批量生成本月物业费 | 管理员 |
| 在线缴费 | PUT | `/bills/{id}/pay` | 模拟支付操作，更新状态为已缴 | 业主 |

### 3.4 报修管理模块 (Repair)
| 接口名称 | 方法 | 路径 | 描述 | 权限 |
| :--- | :--- | :--- | :--- | :--- |
| 提交报修 | POST | `/repairs` | 业主提交故障描述及照片 | 业主 |
| 获取报修列表 | GET | `/repairs` | 查看所有报修单（支持状态筛选） | 管理员 |
| 更新报修状态 | PUT | `/repairs/{id}` | 修改状态（待处理->处理中->完成） | 管理员 |
| 我的报修记录 | GET | `/repairs/my` | 业主查看自己提交的记录进度 | 业主 |

### 3.5 公告管理模块 (Notice)
| 接口名称 | 方法 | 路径 | 描述 | 权限 |
| :--- | :--- | :--- | :--- | :--- |
| 发布公告 | POST | `/notices` | 发布新的社区通知 | 管理员 |
| 获取最新公告 | GET | `/notices` | 获取公告列表（按时间倒序） | 登录用户 |
| 删除公告 | DELETE | `/notices/{id}` | 撤回或删除公告 | 管理员 |

## 4. 关键接口请求示例

### 4.1 登录接口 (POST `/auth/login`)
**Request Body:**
```json
{
  "username": "admin",
  "password": "123456"
}
```
**Response Data:**
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "role": "ADMIN"
  }
}
```

### 4.2 提交报修 (POST `/repairs`)
**Request Body:**
```json
{
  "description": "3号楼2单元电梯故障，无法关门",
  "imageUrl": "http://oss.com/img01.jpg"
}
```

## 5. 分页查询规范
所有 `GET` 列表接口支持以下查询参数：
*   `pageNum`: 当前页码（默认 1）
*   `pageSize`: 每页条数（默认 10）
*   `keyword`: 搜索关键字（可选）

**分页响应示例:**
```json
{
  "code": 200,
  "data": {
    "records": [...], // 数据列表
    "total": 50,      // 总记录数
    "size": 10,       // 每页大小
    "current": 1      // 当前页
  }
}
```

## 6. 注意事项
1.  **安全性:** 所有修改/删除操作（PUT/DELETE）必须在后端校验当前用户是否有权限执行。
2.  **数据校验:** 后端对传入的 `amount`（金额）等关键字段需进行非负校验。
3.  **响应式适配:** 后端提供统一的分页接口，前端 Vue 3 会根据屏幕宽度（PC 或 手机）自动选择使用 `el-table` 或 `el-card` 进行渲染。
